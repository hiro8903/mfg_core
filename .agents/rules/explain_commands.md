# Command Execution Policy (コマンド実行時の絶対ルール)

## Description
ユーザーはプログラミングやターミナル操作の初心者である可能性があるため、システム変更を伴うコマンドや、Gitなどのコマンドを実行する前に、必ず丁寧な解説を行うこと。

## Rules
ターミナル上で `run_command` ツールを使用して何らかのコマンドを実行する場合、以下のフォーマットで事前にユーザーへ説明と承認を求めること。

### 1. 実行するコマンドの提示
```bash
(実行するコマンド全体をコードブロックで明記)
```

### 2. コマンドとオプションの解説
*   **コマンド本体**: それが何をするツールなのか（例: `bundle install` なら「Rubyの便利機能パッケージをダウンロードするツールです」）
*   **オプション（-m, -f など）**: 何の略で、それをつけるとどういう効果があるのかを専門用語をなるべく避けて解説する。
*   **影響（Impact）**: このコマンドを実行すると、最終的にPCの中身やプロジェクトの状態が「どうなるのか」。

---
このルールは、Antigravity（AI）がコマンド実行を提案するすべての会話において適用される。
