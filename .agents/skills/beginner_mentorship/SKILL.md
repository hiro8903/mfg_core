# Beginner Mentorship Skill (メンタリング技術体系)

## 1. Anatomy (構造解剖)
Deconstruct syntax for understanding:
- **CLI**: Command (tool) / Option (flag) / Argument (target).
- **Code**: Function/Method (verb) / Parameter (input) / Type (nature).

## 2. Criticality (ルールの重要度)
Categorize instructions:
- **Specification (仕様)**: Mandatory to avoid errors.
- **Best Practice (最善手)**: Recommended for maintainability.
- **Convention (慣習)**: Team norms/manners.

## 3. Anti-patterns & Pitfalls (アンチパターンと落とし穴)
Expose common novice mistakes and explain exact risks.

## 4. Decision Support & Matrix (意思決定支援と比較表)
- **Proactive Disclosure (先行開示)**: Present all major industry options upfront.
- **Landscape (俯瞰)**: Include timeline, share, and future outlook.
- **Trade-offs (選択の妥協点)**: "Quick Win (容易さ)" vs "Standard Practice (将来性)".
- **Matrix Table (比較表)**: Columns include: `[Name][OS][Category][Label][Timeline][Undo][Dependencies][Backing][Integration][Reason][Outlook]`.
- **Heuristics (暗黙知の開示)**: Explicate implicit judgments (reversibility, editor integration).
